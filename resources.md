## 🧠 Prompt Engineering — NanoBanana (Website Design)

### Iteration #1 — Initial Prompt
You are a professional UI/UX designer.
Create a modern fitness app landing page called "FitForge".
Design requirements:

Hero section with bold headline and CTA button "Start Free Trial"
Navigation: Home, Features, Pricing, Testimonials, Contact
Features section (3 cards): AI Workouts, Nutrition Tracking, Progress Analytics
Testimonials section (2-3 user reviews)
Pricing section (3 plans: Free, Pro $9.99/mo, Elite $19.99/mo)
Footer with social links

Visual style:

Dark background (#0D0D0D) with neon green accent (#39FF14)
Bold modern typography
Mobile-responsive layout
Energetic, motivational feel


### Iteration #2 — Improved Prompt
Improve the previous FitForge design with these additions:

Add a "How It Works" section with 3 numbered steps:
Step 1: Set Your Goal → Step 2: Follow Your AI Plan → Step 3: Track Results
Add a sticky mobile navigation bar at the bottom
(icons: Home, Workouts, Nutrition, Profile)
Add a subtle animated gradient pulse on the hero CTA button
(CSS keyframe animation using the neon green color)
Add a statistics bar between Hero and Features:
"500K+ Users | 10M+ Workouts Logged | 4.9★ Rating | 98% Retention"

Keep all existing sections. Output complete updated HTML.

### 🔄 What Changed Between Iterations

**Iteration #1** established the base structure: color palette, 
sections, and overall layout. It was broad and conceptual.

**Iteration #2** used constraint-based instructions to add 
specific missing elements:
- Added **"How It Works"** section (user journey clarity)
- Added **Stats bar** (social proof / credibility)
- Added **mobile sticky nav** (UX improvement for mobile users)
- Added **CSS pulse animation** on CTA (conversion optimization)

Key technique improvement: switched from general style prompting 
to specific numbered requirements with exact values and behavior 
descriptions. This reduced AI guesswork and produced precise output.
